# Neon Case：赛博探案情景游戏

一个轻量本地 Demo：把 MiroFish 的“多 Agent + 世界状态 + 变量注入 + 未来路径”改造成赛博悬疑互动剧本。

玩家扮演“记忆缺失调查员”，在三幕中选择审问对象、公开线索、观察角色证词和状态变化，最终得到不同真相结局。

## 和 MiroFish 的差异

- MiroFish 预测现实世界如何演化。
- Neon Case 让一个悬疑剧本世界自行演化。
- 玩家不是读固定分支，而是在多 Agent 角色的谎言、信任、嫌疑和隐瞒变化中破案。

## 当前能力

- 一案三幕：零点电梯、记忆黑市、城市预测模型。
- 四名角色 Agent：地下记忆走私者、企业安全主管、城市 AI 接口、前搭档。
- 五个世界状态：真相闭合度、记忆恢复、系统压力、整体信任、系统嫌疑。
- 逐幕推演：第一幕完成后解锁第二幕，第二幕完成后解锁第三幕。
- 动态选项：前一幕的选择会影响后一幕可选行动，每一幕的选项不同。
- 人物关系图谱：角色、死者、玩家、企业和城市 AI 之间用真实连接线展示关系。
- 柱状世界状态：五个状态指标使用柱状图展示，方便对比。
- 多结局：系统共谋、企业凶手落网、误判 AI、与前搭档重启暗线、真相被掩盖。
- SSE 实时事件流：角色发言、状态变化、最终报告逐条推送。

## 故事设定

2097 年，镜城把居民记忆备份接入城市预测模型。Eidolon 首席记忆架构师林栖在死亡前，把一段残缺记忆发给玩家。十分钟后，他死于一场理论上不可能发生的电梯坠落。

玩家扮演记忆缺失调查员：你既要查案，也要追回自己的记忆。每名角色都可能在撒谎，但他们隐瞒的原因不一定是杀人。

四名核心角色：

- 玛拉：地下记忆走私者，掌握黑市备份票据。
- 任舟：Eidolon 企业安全主管，控制案发楼宇日志。
- SORA-9：城市预测 AI 接口，知道系统级异常。
- 岳临：玩家前搭档，保存纸质笔记，也删除过你的定位记录。

## 玩法流程

1. 先阅读案件简报、开场剧情、案发时间线和角色档案。
2. 在“逐幕调查”中选择第一幕行动，点击“推演第 1 幕”。
3. 第一幕推演完成后，系统会解锁第二幕；第二幕完成后解锁第三幕。
4. 每一幕都会基于当前已完成选择和公开线索重新计算世界状态。
5. 第三幕完成后生成真相报告和结局。

## 运行

推荐直接双击：

```text
D:\AI学习\Hackathon\newdemo\start-neon-case.bat
```

它会启动本地服务并自动打开浏览器。

也可以手动运行：

```bash
cd /d D:\AI学习\Hackathon\newdemo
node server.js
```

然后访问：

```text
http://127.0.0.1:8797
```

如需关闭服务，双击：

```text
D:\AI学习\Hackathon\newdemo\stop-neon-case.bat
```

## 部署到 Vercel
