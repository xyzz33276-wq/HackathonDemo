Problem

内容创作最贵的不是写作，而是上线后的试错

短剧和网文的商业结果高度依赖节奏、钩子、爽点兑现和付费卡点。现在大量判断仍依赖经验：写完、上线、看数据，再回头改，成本高且反馈滞后。

我们的目标不是替作者写完一切，而是在关键分幕前提供一个“虚拟观众试映场”。

01

下一幕怎么写，缺少低成本验证

同一个设定可以走反杀、误会、揭密或暂缓路线，但作者很难在上线前知道哪条路线更能推动追更。

02

爆款判断主观，难沉淀成团队资产

爽点、虐点、付费点常被口头讨论，缺少结构化评分、分歧原因和可复盘 Trace。

03

平台反馈滞后，改稿窗口很短

完读率、评论和付费数据往往在发布后才出现，真正需要的是投放前的风险预警和改稿建议。

一句话：不是问 AI “这个剧本好不好”，而是让剧本先进入一组虚拟观众和平台 Agent 的试映场，提前看到它会在哪里追更、付费、吐槽或弃坑。

Current Demo

Neon Case：可玩的多 Agent 剧情沙盘

`情景式游戏` 不是静态互动小说，而是一个可以演化的悬疑世界。玩家每一幕的选择都会影响角色证词、世界状态、可选路线和最终结局。

这个 Demo 证明了最小闭环：剧本可以被拆成角色、状态、线索、选择和报告。

先让听众认人：五张角色卡对应五条剧情力量

Neon Case 的演示重点不是“谁长得像凶手”，而是每个角色的隐瞒都会推动世界状态变化。角色图让现场观众快速把证词、动机和路线对应起来。

Act System

一案三幕推演

第一幕找谎言，第二幕判断目的，第三幕选择公开口径。结构适合扩展成分幕续写。

Agent Cast

角色 Agent 驱动

任舷、玛拉、SORA-9、岳临各有动机、隐瞒和信任变化，不只是固定台词。

World State

五类世界状态

真相闭合度、记忆恢复、系统压力、整体信任、系统嫌疑，让剧情变化可视化。

Dynamic Choice

动态选项解锁

上一幕选择会影响下一幕可选行动，形成可解释的剧情路径，而非简单树状分支。

Live Stream

3 分钟看懂 Demo 玩法

现场讲解时可以直接按这条线走：玩家不是阅读固定分支，而是在每一幕选择调查行动，观察 Agent 证词与世界状态如何变化，最后决定公开哪一种真相。

1读案件档案

明确玩家身份、受害者林栖、四名嫌疑角色和核心规则：第一个撒谎的人不一定是凶手。

2选择公开线索

电梯日志、记忆碎片、黑市票据、纸质笔记会影响真相、压力、信任和系统嫌疑。

3逐幕选择行动

每一幕只能选一个动作，例如强压任舷、追问 SORA-9、相信岳临或保护玛拉。

4看实时推演流

角色会发言，状态会变化，系统会提示本幕发现证明了什么、又遮住了什么。

5生成真相报告

第三幕后得到多结局报告：系统共谋、企业落网、误判 AI、重启暗线或真相被掩盖。

荐 路演推荐路线：稳健破案线

这条路线最容易让听众理解反转：先抓住最像凶手的人，再证明真正危险的是系统。

  • A1
    第一幕：强压任舷交出二级日志。
  • A2
    第二幕：审计巡逻机器路径，把案件推向城市网络。
  • F
    第三幕：揭露城市预测模型共谋，生成系统共谋结局。

Product Engine

下一步：Drama Smallville 剧本试映风洞

`script simulation` 已经探索出下一代商业化引擎：输入剧情材料，生成候选续写，让观众 Agent 和平台 Agent 进行两轮评价，最后输出推荐方案、置信度、关键分歧和改稿建议。

Neon Case 证明“剧情可推演”，Drama Smallville 把它变成“剧本可试映、可评分、可改稿”。

STEP 1剧情材料

角色设定、前情梗概、当前稿、作者意图和约束。

STEP 2候选续写

生成多个下一幕方向，对齐作者保留元素和禁用元素。

STEP 3观众试映

爽点、情感、付费、平台推流 Agent 分别判断。

STEP 4加权评分

留存、好评、付费、平台推荐和人设一致性聚合。

STEP 5改稿报告

输出推荐分支、分歧点、弃坑风险和下一轮 prompt。

Business Value

不是通用写作助手，而是内容团队的爆款验证工作台

产品价值来自“预测前置 + 改稿闭环 + 私有数据校准”。它不替代作者的判断，而是把团队经验、观众反应和平台目标变成可复盘的决策系统。

商业表达聚焦短剧团队、网文作者、MCN 和内容工作室，不使用未经验证的市场规模数字。

For Writers

降低改稿焦虑

作者可以输入当前剧情和创作约束，快速比较多个下一幕方向，看到哪条路线更适合留存、追更或付费。

  • 保留作者意图，不强行替作者决定。
  • 输出可执行改稿建议，而不是泛泛点评。
For Studios

提高选题和分集决策效率

短剧团队可以在拍摄前测试关键付费卡点、反转可信度和平台适配风险，减少后期重拍和投流浪费。

  • 适合多版本 A/B 剧情试映。
  • 适合复盘爆款案例和失败样本。
For Platforms

沉淀私有观众模型

随着真实完读、评论、付费数据接入，客户可以形成自己的观众偏好、平台权重和题材记忆。

  • 支持不同平台、题材、目标的评分权重。
  • Trace 和 Memory 能形成长期数据壁垒。
SaaS 订阅面向个人作者和小型工作室，按月使用虚拟试播能力。
按次试映按剧本、分集或候选方案计费，适合项目制团队。
团队版支持项目历史、多人协作、Trace 复盘和私有模板。
API / 私有化接入平台反馈数据，训练客户专属观众校准模型。

Roadmap

从可玩 Demo 到可商业化引擎

路线图以“先可演示，再可解释，再可校准”为原则。P0 不追求百万 Agent，而是把剧情试映闭环打通；P1 用统计拟合提升可信度；P2 接入真实数据形成壁垒。

核心原则：真实宏观数据 -> 数学分布 -> 4-7 个代表 Agent -> 拟合校验 -> 轻量模拟。

P0

可玩 Demo + 虚拟试播闭环

证明剧情可以被状态变量和多 Agent 推演，也证明剧本可以被候选续写、观众试映和报告系统评估。

已完成 / 已探索
  • Neon Case 三幕互动推演。
  • 角色 Agent、世界状态、多结局报告。
  • Drama Smallville simple-run 试播接口。
  • 留存、好评、付费、平台推荐评分。
P1

统计拟合观众 Agent

从手写人设升级为统计约束的代表性观众，前端可查看 Agent 画像、权重、阈值、拟合误差和 Trace。

近期产品化
  • 行业统计初始化虚拟观众分布。
  • 压缩为 4-7 个代表 Agent。
  • Trace、Memory、Population Fit 可视化。
  • 支持短剧、网文、平台目标切换。
P2

真实反馈校准与私有模型

接入客户历史项目数据，把预测结果和真实完读率、评论、付费转化对齐，形成持续学习的内容决策资产。

商业化阶段
  • 导入完读率、追更率、评论关键词、付费率。
  • 校准平台权重和 Agent 行为阈值。
  • 建立客户私有爆款案例库。
  • 支持团队版报告、API 和私有化部署。

路演收束句

Neon Case Studio 的价值不是“再做一个互动故事”，而是把互动故事背后的多 Agent 推演能力，升级成短剧和网文创作的试映基础设施：让爆款在上线前先被模拟、被质疑、被打分、被改好。
